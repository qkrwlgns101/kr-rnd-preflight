# 보안 정책

## 데이터 처리 방식

- 앱은 사용자의 PC에서 `127.0.0.1`에만 임시 웹 서버를 엽니다.
- 실행할 때마다 임의의 48자리 경로 토큰을 생성합니다.
- 선택한 CSV와 검사 결과를 외부로 전송하지 않습니다.
- 텔레메트리와 사용자 계정 기능이 없습니다.
- 입력 본문은 최대 25MB이며, 처리 후 별도 파일로 저장하지 않습니다.
- 30분 동안 요청이 없으면 프로세스를 자동 종료합니다.

이 설계는 신뢰할 수 있는 개인용 Windows PC에서의 로컬 사용을 전제로 합니다. 악성 프로그램에 이미 감염된 PC, 여러 사람이 공동 사용하는 계정 또는 원격 데스크톱 환경의 기밀성까지 보장하지 않습니다.

## 민감정보 주의

실제 집행내역, 주민등록번호, 계좌번호, 급여자료, 증빙파일 또는 접근 토큰을 GitHub Issue나 Pull Request에 올리지 마세요. 재현 자료가 필요하면 실제 값과 구조를 충분히 바꾼 최소 가상 데이터만 사용하세요.

## 취약점 제보

민감정보가 포함되지 않은 일반 버그는 GitHub Issue로 제보할 수 있습니다. 공개하면 악용될 수 있는 보안 취약점은 저장소의 Security 탭에서 비공개 보안 제보를 사용하세요. 실제 고객자료나 개인정보는 제보에 첨부하지 마세요.

## 실행 파일 서명

`v0.1.0` 실행 파일은 코드 서명 인증서로 서명되지 않았습니다. GitHub Release에서만 내려받고 함께 제공되는 `SHA256SUMS.txt`와 로컬 파일의 SHA-256을 비교하세요.
