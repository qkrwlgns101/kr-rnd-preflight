# 검사 규칙과 한계

확인일: 2026-07-19

이 문서의 규칙은 공식 규정을 코드로 완전히 구현한 것이 아닙니다. 사용자가 입력한 표들 사이에서 재확인이 필요한 후보를 찾기 위한 보수적인 데이터 일관성 검사입니다.

| 규칙 ID | 표시 조건 | 위험도 | 주요 한계 |
|---|---|---:|---|
| `R001_DATE_OUTSIDE_PERIOD` | 취소되지 않은 거래의 집행일이 입력한 현재 협약기간 밖임 | 높음 | 예외 승인이나 별도 인정기간을 판단하지 않음 |
| `R002_DUPLICATE_INVOICE` | 취소되지 않은 거래에서 같은 증빙번호가 반복됨 | 높음 | 정당한 분할 집행 여부를 판단하지 않음 |
| `R003_EVIDENCE_MISSING` | 증빙 ID가 공란이거나 증빙목록에 없음 | 높음 | 원본 증빙의 진위·충분성을 판단하지 않음 |
| `R004_BUDGET_CODE_UNKNOWN` | 집행 비목코드가 예산표에 없음 | 보통 | 비목 변경 승인 여부를 판단하지 않음 |
| `R005_BUDGET_EXCEEDED` | 날짜순 누적 집행액이 비목예산을 처음 넘음 | 높음 | 현물·이월·이자·환입 등 별도 정산 논리를 완전히 반영하지 않음 |
| `R006_PARTICIPANT_UNKNOWN` | 인건비 코드 거래의 참여인력이 목록에 없음 | 보통 | 참여기간, 인건비율, 급여 적정성을 판단하지 않음 |
| `R007_VENDOR_NAME_INCONSISTENT` | 동일 거래처 ID에 다른 이름이 사용됨 | 낮음 | 상호변경, 띄어쓰기 등 정당한 차이도 표시할 수 있음 |

## 계산 원칙

- 날짜는 시작일과 종료일을 포함합니다.
- 상태가 `취소`, `취소됨`, `환입`, `cancelled`, `canceled`인 행은 검사 대상과 예산 누적에서 제외합니다.
- 예산 누적은 집행일자, 원본 CSV 행 순서로 계산합니다.
- 거래 ID가 없으면 CSV 데이터 순서에 따라 `ROW-0001` 형식의 임시 ID를 부여합니다.
- 금액은 쉼표, `₩`, `원`, 괄호 음수를 읽을 수 있습니다.
- 날짜는 `YYYY-MM-DD`, `YYYY.MM.DD`, `YYYY/MM/DD` 및 Excel 날짜 일련번호를 읽을 수 있습니다.
